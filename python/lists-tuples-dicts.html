Python: Lists, Tuples, and Dictionaries

Learning the differences between all three

Introduction

Many beginners that are initially delving into the Python coding language get stuck on storing data when making games, apps, or other useful projects in it. There are three major ways to do this that are common to most coders: lists, tuples, and dictionaries. Since they can get a tad bit complicated, we'll jump right in to covering the basic principles of each, and how to use them.

Lists

Lists are mostly self-explanitory: they will hold however many items you want to add in a list format. One very useful aspect to a list is that they are are modifiable (or 'mutable', to be coding-savvy), allowing the user to change the order of items, add more items, and etcetera. They are created simply by typing a name for the list, then setting it equal to your list items inside a pair of square brackets.

ice_cream_flavors = ['Vanilla', 'Mint', 'Chocolate', 'Coffee']

Inside our list, we added four individual strings, which are our current list items. The other possible items that can be added to any list are integers. No matter what the item, each value must be seperated by a comma.

Rule of Thumb: Python, along with many coding languages, counts starting from zero. So in the example above, we users see four items, but Python reads it as three (Vanilla 0, Mint 1, Chocolate 2, Coffee 3). This is true for everything in Python, so it is important to keep in mind!

Tuples

Tuples are very similar to lists when it comes to storing data, but have one major difference. They are immutable, meaning they cannot be changed ever. How would this ever come in to play? Can't we just use lists instead? While it is true that in most cases you will use lists, there are a few scenarios where tuples come in handy. Say you've created a game where you want to store the players hitpoints. In case you aren't a gamer, know that this is a number you don't want jumping around and changing in most cases. Here's where you'd use tuples instead of a list; by storing it this way the numbers will stay consistent and secure. Tuples are created by simply using parentheses and listing your strings or integers inside.

days = ('Sunday', 'Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday')

Dicionaries

Dicionaries are much different than lists or tuples, although they are often grouped together with them. Their defining compenent is what's called a key. Inside each key is stored a value. To create this, we use curly braces. A profile is a good example dictionary.

profile = {'Name': 'Jerry', 'Age': 37, 'Occupation': 'Painter'}

As you can see, each key (the 'Name', 'Age', and 'Occupation') is seperated with a colon, and we distinguish the key and value with a comma at the end. The values can even be left blank, allowing the user to enter in his or her desired information. After that, we can call back the dictionary and print the key without knowing anything about the value. Let's see what printing a key inside a dictionary looks like.

>>> profile['Age']
37

Calling the key is elementary Python. Simply type in the dictionary name, then enter in whatever key you want inside square brackets.

Conclusion

Speaking from personal experience, I have found lists the most helpful in my coding, but there have certainly been uses for all three of them in my years of coding. Now that you know the distinctions between them, I'm sure you will find many uses for them as well!
