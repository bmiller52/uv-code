Python: Intro to Functions

Learning the basics of functions and keeping your code clean

Introduction

Lets start off by what functions are. A function is a block of code that can be reused however many times as you want. Python comes with predefined functions like print() and raw_input(). But we can still make our own functions to make our code cleaner and more compact. We do this by writing def, which is short for define, then your function name followed by opening and closing parentheses and a colon.

def profile_setup():
  question1 = raw_input('Hello, what is your name? ')
  question2 = raw_input('How old are you? ')
  question3 = raw_input('What\'s your job? ')

  print()   print('Name: ' + question1)
  print('Age: ' + question2)
  print('Job: ' + question3)

Now if you run this code in a script or the Python console, nothing will happen. That's because we only defined it but never called it. To do this we just type the function name with the parentheses.

profile_setup()

Now if we call it, it will prompt use the three questions, then output the responses. This is really handy when you want to use this multiple times. So now, you only need to type the code out once, but it can be used however many times you need.

Going Further with Functions

But what if we want the user to enter in something before we run the function to determine the output of the function. Here's a real simple example.

def do_math(x, y):
  calculation = x + y
  return calculation

num1 = int(raw_input('Pick any number '))
num2 = int(raw_input('Pick another number '))

do_math(num1, num2)

If your run this script, the user will determine to two numbers that the function will add. What we put inside the parentheses when we define the function, is arbitrary. They are essentially varibles. Although, notice the int function that is outside of the raw_input. This program wouldn't work without it. It just parses whatever the user inputs to be an integer. If we didn't have this, it would look at it as a string and would concatenate it. So it wouldn't acually add the numbers.

Conclusion

Functions are very helpful esspesially when you code DRY (Don't Repeat Yourself). Many times when you make games or just any program, you're going to need functions to keep your code clean and DRY.
