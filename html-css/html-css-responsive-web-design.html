HTML/CSS: Responsive Web Design

What is responsive web design and how do we correctly use it?

What is it?

Responsive web design is simply webpages that look good on any screen or device. Meaning that images, paragraghs, and other elements shrink, move, or enlarge depending on how big the screen size is. This wasn't a proplem for a while until phones became more popular. Here's a graph of all website data viewed on moblie devices from 2009 to 2017.

As you can see, making your website responsive to moblie is very important. You propley have seen some sites that have extremly small text and you have to zoom in to see it. That's a prime example of a poorly designed website. Infact developers now are taught to first make the website on moblie then adjust it to desktops or laptops.

For instence, here's a page you made on your desktop.

But this is what it looks like on moblie

Now the user has to scroll horizontaly over to see the entire page. This page looks ugly and often leaves your customers frustated and ther're unlikly to come back. Let's see a good responsive page on moblie now.

Now it's one continuous page scrolling down. This is much easier to read the information and your viwers will be happy. How do we acually code this? Here's a simple example for you to try out.

<!DOCTYPE html>
<html lang="en-us">
<head>
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<style>
.box {
  float: left;
  margin: 5px;
  padding: 15px;
  max-width: 300px;
  height: 300px;
  border: 1px solid black;
}
</style>
</head>
<body>

<h1>Responsive Web Design Example</h1>
<h2>Try resizing your browser window</h2>

<div class="box">
  <h2>Red</h2>
  <p>Red is a prime color. Did you know that?</p>
  <p>It can also be the color of an apple, and a shinny new bike</p>
</div>

<div class="box">
  <h2>Green</h2>
  <p>Green is the second prime color.</p>
  <p>Green is the color of trees and grass in the spring time.</p>
</div>
<div class="box">
  <h2>Blue</h2>
  <p>Blue is the last prime color. Have you seen RGB?</p>
  <p>It stands for Red, Green, Blue. </p>
</div>

<div class="box">
  <h2>Yellow</h2>
  <p>This is not a prime color.</p>
  <p>You can make Yellow by mixing Red and Green.</p>
</div>

</body>
</html>

As you can see, all of the magic is done in the CSS. The first rule is float: left;. It's saying for each box, at all times, push it as far to the left as posible. We then add margin: 5px;. We don't specify which side so it will add 5 pixels of margin all around the element. The same goes for the padding. The next two lines are just setting how big the box should be. And the last line adds a thin line all around the boxes.

Responsive web design, like in this example, can be really simple. But in other situations, it can get quite complex. Sometimes if you have a lot of content on the page and javascript, you have to tell the code to check to see how big the screen size is, then adjust accordingly. This will add much more code to write and will slow down your coding process. Another example is say you have a sidebar when your viewing the page on a desktop, but when someone sees it on a phone, you want that xtra context to hide and instead, have a button. Responsive web design is extremly important since phones are becoming more popular every day.
