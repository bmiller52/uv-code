HTML/CSS: Margin and Padding

Comparing their qualities and appropriate uses

Margin vs. Padding

One of the most common struggles with CSS is determining how and when to use margin or padding. As a beginner to HTML and CSS this definitely confused me, and was much easier to understand with a visual demonstration to explain it. Let's take a look at it with the Box Model.


Margin seperates the entire element from all of the other elements. In other words, it generates the white space outside the border as seen in the Box Model.

Padding pushes the border away from the content. The content can be text, an image, etc.

Application

When using just margin: ; or padding: ; in a CSS property, four specifications are necessary. Starting from the top, the four margin or padding attributes work their way around clockwise like so--margin: top, right, bottom, left; and padding: top, right, bottom, left;. Using the margin as the example here, this is what it should look like in the actual CSS code,:

margin: 40px, 10px, 40px, 10px;

This method will add all of the margins for each side at once in one property (40 pixels of margin to the top and bottom, and 10 pixels to the right and left). In addition to this, there's another method that will do the same thing, only specifying whatever individual side the margin is being added to. As seen below, there are four different properties for the four different sides.

  • margin-top
  • margin-right
  • margin-bottom
  • margin-left

Each of these only requires one attribute, for example:

margin-top: 40px;

Similarly, all the same rules apply to padding as well.

Examples

To further demonstrate and recap margin and padding, let's look at several examples.

Remember: If you only apply one attribute when you use simply margin or padding, it will set that number for all four sides as in the example below (starting with the margin).

Code

div {
  border: solid 2px green;
  margin: 50px;
}

Output

This example has 50 pixels of margin all around.

Output with no margin

This example has 0 pixels of margin all around.


Padding Examples

In the same way, we can demonstrate padding in a similar example.

Code

div {
  border: solid 2px green;
  padding: 50px;
}

Output

This example has 50 pixels of padding all around.

Output with no margin

This example has 0 pixels of padding all around.


Conclusion

The Box Model is a useful tool to remember for CSS, especially when differentiating between margin and padding. However, these properties can go far beyond the Box Model, and with the correct knowledge, they will be beneficial in advanced coding. Both properties are practically essential in websites, allowing for a clean, modern look that demonstrates professionality and expertise.
