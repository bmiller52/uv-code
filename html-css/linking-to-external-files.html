HTML/CSS: Linking to External Files

Learning how to be much more time efficient with HTML

Introduction

In a lot of basic teaching on HTML and making webpages, a beginner procedure is to put all of your CSS and JavaScript code in the one big file together. Many courses will do this because it's simply easier to have everything in the same file, avoiding switching back and forth between different ones and getting lost. While there are a few perks to this method of coding, it often can become convoluted and messy once you have about twenty different pages on your website. That makes you have to change the code in every single page file in order to change any formatting style for your whole website, and who has time for that? There are definitely many smoother ways to connect your CSS and JS to the main HTML file, and external files is a really helpful solution we like to use for this. The concept is simple: split up each individual language into their own files, creating a dominant file for each type of code that can be easily connected to your HTML. This way, when you change one element of styling in the CSS file, it will change among all of the HTML files linked to it. The same concept applies to JavaScript. Here's an example to link to a CSS file:

<link href="css/main.css" rel="stylesheet">

Unlike most other HTML tags, link is a "stand alone", meaning it only has an opening tag and no close. Right after this, the href attribute stands for Hyperlink Reference; this is a key element here as it's allowing us to actually go about linking whatever external file we want to reference our main file to. Inside the quotes is the path to wherever your file is stored. In this example, it's path is to go to the folder named css, then find the file named main.css inside that folder. If there are multiple folders the file is stored inside, you simply list the broadest folder first, and work your way down with each slash until you get there. The final part is the rel attribute, which specifies the relationship between the current document and the linked document. For a CSS file we use stylesheet.

Now that we've made the tag, we need to place it correctly. The best line to insert it in for most code is inside the head tags. Here is the most convenient spot to put it as everything inside the head loads even before the page itself does. It gets first priority as opposed to the body tags, which get loaded afterwards. Since we want to load the styling before the main content of the page gets there, this is important to remember. The process for linking to a JS file is a bit different, but no less easy:

<script src="js/scripts.js"></script>

For the JavaScript to work we use script tags (which do have an opening and close). Instead of using a href, JS requires src, which quite simply means source. Making the path to the file is exactly the same as before, only we want our JavaScript file instead of our CS this time. There are varying opinions on where to put the script tag--some put it in the head with their CS file, but I would recommend putting it at the very end of the code, right before the closing body tag. While it's not necessary, it is pretty useful, as when you try to load the JS before the rest of the page, it can take a while to get to the actual content of the page.

If this is done correctly, you now have a master control with all of your CSS and JavaScript tidily stored away in their own files but still just as functional to your website.
